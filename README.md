📦 RetailCRM to Supabase Pipeline & Dashboard
Проект по автоматизации обработки заказов: импорт данных в RetailCRM, синхронизация с облачной базой данных Supabase, визуализация на Vercel и уведомления в Telegram.

🚀 Стек технологий
Backend: Python (Requests, Pydantic, Logging)
Database: Supabase (PostgreSQL)
Frontend & Deployment: Next.js (v0.dev), Vercel
Notifications: Telegram Bot API

🤖 Использование AI (Prompt Engineering)
В ходе разработки я активно использовал AI для ускорения написания кода и генерации фронтенда. Основные промпты:
Для бэкенда: "Напиши функцию на Python для пакетной загрузки заказов (метод /orders/upload) в RetailCRM с использованием сессий и таймаутов."
Для валидации: "Создай строгую модель Pydantic для валидации заказа из RetailCRM. Если externalId отсутствует, сгенерируй его на основе телефона и имени клиента."
Для фронтенда (v0.dev): "Создай дашборд на русском языке с графиком заказов. Данные бери из Supabase (таблица orders_crm, колонки total_sum и created_at). Валюта — тенге (₸)."
Для уведомлений: "Напиши функцию проверки суммы заказа в цикле и отправки сообщения в Telegram, если сумма > 50,000 ₸."
🛠 Где я застрял и как решил (Troubleshooting)

1. Ошибки API RetailCRM (Error 460)
Проблема: При попытке загрузки заказов сервер возвращал загадочную ошибку 460 Client Error.
Решение: С помощью расширенного логирования и вывода response.text выяснилось две причины:
Отсутствие обязательного externalId. Решено через генерацию ID в Pydantic.
Использование несуществующего orderType ("eshop-individual"). Исправлено на валидный код "main" через @field_validator.

2. Проблема "Нулевых сумм"
Проблема: В базу Supabase заказы попадали с total_sum = 0.
Решение: Проанализировал ключи JSON через функцию analyze_json_keys. Выяснилось, что в текущей версии API RetailCRM поле называется totalSumm (с двумя 'm'), а не totalSum. Исправил alias в модели Pydantic.

3. Кэширование переменных в Jupyter
Проблема: Обновление токенов в файле .env не подхватывалось кодом.
Решение: Использовал load_dotenv(override=True).

🏆 Самое сложное: Настройка Vercel и связка с Supabase
Самым сложным этапом оказалась настройка деплоя на Vercel и его интеграция с Supabase.  
Трудности:  
Фантомные проекты: Автоматическая интеграция Vercel создала новый пустой проект в Supabase (emerald-notebook) вместо того, чтобы подключиться к моему существующему. Это привело к тому, что сайт показывал пустые графики.
Environment Variables: Переменные, созданные интеграцией (SUPABASE_URL), не подхватывались Next.js-приложением.
Mock Data: Инструмент v0 зашивал в код тестовые данные (заглушки), которые перекрывали реальные данные из базы.

Как решил:  
Вручную перепривязал проект в настройках интеграции Vercel к правильному ID Supabase.
Вручную переименовал и перепроверил все переменные окружения в панели Vercel.
Добавил политику RLS (Row Level Security) в Supabase, чтобы разрешить публичное чтение (select) для роли anon.
Через промпты в v0 заставил ИИ полностью удалить "mock data" и использовать только fetch из API.

# --- RetailCRM Настройки ---
SUBDOMAIN=ваш_поддомен          # Например: shop-123 (из адреса shop-123.retailcrm.ru)
API_KEY=ваш_ключ_crm            # Настройки -> Интеграция -> Ключи API
SITE=ваш_код_магазина           # Настройки -> Справочники -> Магазины (Символьный код)

# --- Supabase Настройки ---
SUPABASE_URL=https://id.supabase.co
SUPABASE_SECRET_KEY=ваш_role_key # Project Settings -> API -> service_role (секретный ключ)
PROJECT_ID=ваш_id_проекта        # Тот самый код из 20 символов из URL Supabase
PROJECT_NAME=название_проекта

# --- Telegram Настройки ---
TG_TOKEN=730795...              # Получен от @BotFather
TG_CHAT_ID=12345678...          # Получен от @userinfobot

Где взять эти данные (Гайд):
RetailCRM:
SUBDOMAIN: Это имя перед .retailcrm.ru.
API_KEY: В CRM зайдите в Настройки -> Ключи API. Создайте ключ с правами на чтение/запись заказов.
SITE: Зайдите в Справочники -> Магазины. Нужен именно «Символьный код».
Supabase:
URL и Secret Key: Зайдите в свой проект -> Project Settings (шестеренка) -> API.
ВАЖНО: Для Python-скрипта используйте service_role (Secret Key), а для настроек Vercel используйте anon (Public Key).
PROJECT_ID: Виден в настройках General или в URL вашего браузера, когда вы в панели Supabase.
Telegram:
TG_TOKEN: Напишите /mybots боту @BotFather.
TG_CHAT_ID: Напишите любое сообщение боту @userinfobot, он вернет ваш персональный ID.
